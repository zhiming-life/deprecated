# 构建思路：怎么搭一个命理 AI

[zhiming.life](https://zhiming.life) 的架构思路，开源版代码是它的最小可运行缩影。读完回看 `conductor.py`，就懂每个文件为什么这么写。

## 一、窄而深

什么都做一点（八字 / 六爻 / 紫微 / 风水……）的结果是每样都浅、还互相打架。我们的选择是收敛核心、把深度做透。这个 demo 只演示八字一条线。

## 二、三层架构：分诊 / 算法 / 解读

```
用户 → Supervisor（问诊·判意图·路由）→ 确定性算法层（排盘，只输客观事实）→ Specialist（在盘面上解读）
```

对应 `agents/supervisor.py` → `core/paipan()` → `agents/bazi_master.py`，由 `conductor.py` 串起来。

- **为什么拆开分诊与解读？** 让专家 prompt 专注推理，不被收信息的逻辑稀释。
- **为什么要确定性算法层？** 排盘是硬逻辑（历法、节气），交给 LLM 会算错。盘面用 Python 算死再交给师傅——这条接缝是质量地基。

## 三、关键边界：算法只给事实

排盘算法只输出客观的干支与五行关系，判断交给解读层。`core/wuxing.py` 演示这条边界。

## 四、解读的形：结构化分段 + 标注把握程度

与其让 LLM 自由发挥一大段，不如分段递进，并对每个判断标注把握程度（高 / 中 / 低）——既诚实，也让用户分得清哪些能信。段名自己定，见 `prompts/bazi_master.py`。

## 五、有据可查

让解读有出处靠 RAG——给师傅投喂相关的参考资料（只给接口，自己接语料）。参考资料只用于触发联想，不作判决依据。
